O presente trabalho consistiu no desenvolvimento de uma aplicação web dinâmica utilizando o micro-framework Flask para a gestão e visualização de um dicionário de terminologia médica. A arquitetura da aplicação assenta no modelo cliente-servidor, onde o backend, processado em Python, é responsável pela gestão das rotas e pela manipulação dos dados armazenados num ficheiro em formato JSON (dicionario_medico.json).

No que concerne à lógica do servidor, o script aula7_web.py inicia o carregamento da base de dados para a memória através da biblioteca json, permitindo que o acesso aos conceitos seja eficiente. Foram implementadas quatro rotas principais: uma página inicial (/), uma listagem completa de conceitos (/conceitos), uma rota de detalhe dinâmica (/conceitos/<designacao>) e um endpoint de API (/api/conceitos). Esta última permite a interoperabilidade do sistema, disponibilizando os dados estruturados para consumo por outras máquinas ou serviços.

A interface de utilizador (frontend) foi construída recorrendo ao motor de templates Jinja2, que permite a geração dinâmica de conteúdo HTML. Para garantir uma experiência de utilizador moderna e responsiva, integrou-se a framework Bootstrap 5 através de um ficheiro de base, o layout.html. Este ficheiro estabelece a estrutura comum a todas as páginas, incluindo a importação de estilos e scripts externos, bem como a definição de blocos que são preenchidos especificamente por cada página secundária, como a home.html ou a conceitos.html.

A navegação entre termos médicos é realizada de forma automática: a aplicação percorre as chaves do dicionário e gera hiperligações que direcionam o utilizador para a definição correspondente. Caso o utilizador tente aceder a um termo inexistente, o sistema está preparado para capturar essa exceção e apresentar uma página de erro personalizada, garantindo a robustez do software. Em suma, o projeto demonstra a integração prática entre processamento de dados estruturados e a sua exposição num ambiente web moderno e funcional.
