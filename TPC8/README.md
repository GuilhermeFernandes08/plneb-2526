O projeto desenvolvido consiste numa plataforma web completa para a consulta de um dicionário médico, estruturada sobre o framework Flask e alimentada por uma base de dados em formato JSON. A arquitetura da aplicação baseia-se na separação entre a lógica de servidor (backend) e a apresentação visual (frontend), permitindo que o sistema seja simultaneamente robusto no processamento de dados e flexível na interface com o utilizador. No backend, o ficheiro aula7_web.py gere o ciclo de vida da aplicação, desde o carregamento inicial dos dados médicos até à definição das rotas que controlam o fluxo de navegação.

A estrutura visual do site utiliza o conceito de Template Inheritance do motor Jinja2, centralizado num ficheiro layout.html que carrega a framework Bootstrap 5. Este ficheiro define os elementos persistentes, como uma barra de navegação responsiva e estilizada, que permite ao utilizador transitar rapidamente entre a página inicial, a listagem completa de termos e a ferramenta de pesquisa avançada. A utilização de blocos dinâmicos garante que cada página secundária herde o design profissional do Bootstrap, mantendo a consistência estética em todo o domínio.

O sistema de navegação de conceitos foi desenhado para ser intuitivo e dinâmico. Na página de listagem, o Flask percorre todas as chaves do dicionário médico para gerar automaticamente links clicáveis, que direcionam o utilizador para rotas variáveis. Estas rotas dinâmicas capturam o termo selecionado na URL e filtram a base de dados em tempo real para apresentar a definição exata na página de detalhe. Caso o utilizador tente aceder a um termo inexistente ou ocorra um erro de navegação, o servidor está programado para renderizar uma página de erro personalizada, garantindo a continuidade da experiência de utilização.

Para além da consulta direta, a aplicação integra uma funcionalidade de pesquisa avançada que utiliza expressões regulares para oferecer filtros de precisão. O utilizador pode optar por pesquisas que respeitam a capitalização de letras ou que procuram apenas por palavras exatas, evitando resultados ambíguos. Quando são encontrados resultados, o servidor processa o texto para inserir destaques visuais em negrito, que são renderizados com segurança no navegador através do filtro | safe. Por fim, a aplicação disponibiliza também um endpoint de API, que expõe o dicionário completo em formato JSON para integração com outros sistemas, demonstrando a versatilidade do projeto tanto para utilizadores humanos como para consumo automatizado de dados.
